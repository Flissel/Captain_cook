# Offline demo

The offline demo runs the actual event-driven supply-chain pipeline with a
deterministic decomposer and four dependency-free Householder workers:
Architect, Ledger Steward, Delivery Builder, and Quality Warden. Each emits a
structured offline report that names its prompt artifact, evidence source, and
limitation. `EchoWorker` remains an integration-test fixture; it is not the
judge-facing demo worker.

```powershell
python main.py demo --output artifacts/demo-run.json
```

No API key, network connection, browser, Docker daemon, or external service is required.

## Evidence artifact

`artifacts/demo-run.json` is regenerated by the command above. It contains:

- `success`: true only when all four expected role tasks reached `done` and the ledger recorder reported no errors;
- `problem_id`: a fresh ID for the submitted root problem;
- `terminal_count` and `done_count`: lifecycle totals from the ledger read side;
- `blocks`: JSON-safe summaries of the genesis, problem, and subproblem ledger records.

Inspect it with:

```powershell
Get-Content artifacts/demo-run.json | ConvertFrom-Json | Format-List
```

The artifact is evidence of the current vertical slice. It does not simulate or
claim Hermes, Codex CLI execution, n8n deployment, MariaDB, Minibook
integration, or live LLM/MCP execution by the Householder workers.
